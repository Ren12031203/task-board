# 症例タスクボード — GitHub Pages 公開手順

`index.html` を GitHub Pages に置くと、固定URLのWebアプリになります。
**URL（保存先）が固定されるので、アプリを更新しても症例データは消えません。**
スマホからも同じURLで開けます。

> 重要：GitHubに上がるのは**プログラム（index.html）だけ**です。
> 症例データ（動物名・飼い主名など）は各自の端末内（ブラウザ）にだけ保存され、リポジトリには一切載りません。

---

## 手順（5分）

### 1. リポジトリを作る
1. GitHub にログイン → 右上「＋」→ **New repository**
2. Repository name：`case-task-board`（任意）
3. 公開範囲：**Public** を選ぶ
   - 無料アカウントの Pages は Public が必要です（コードは公開されますが、患者データは含まれません）
   - どうしても非公開にしたい場合は GitHub Pro 等が必要です
4. **Create repository**

### 2. index.html をアップロード
1. リポジトリ画面で **Add file** → **Upload files**
2. `index.html` をドラッグ＆ドロップ
3. 下の **Commit changes** を押す

### 3. Pages を有効化
1. リポジトリの **Settings** → 左メニュー **Pages**
2. Source：**Deploy from a branch**
3. Branch：**main** ／ フォルダ：**/(root)** → **Save**
4. 1分ほど待つと、上部に公開URLが表示されます
   - `https://<ユーザー名>.github.io/case-task-board/`

### 4. 開いてブックマーク
- 上記URLをPC・スマホのブラウザで開く
- 初回は「はじめに」画面が出るので、手元の**バックアップJSONを読み込む**（または新規開始）
- 以降はこのURLでデータが保持されます。ブックマーク／ホーム画面に追加しておくと便利

---

## 更新のしかた（今後アプリを直したとき）
1. リポジトリで `index.html` を開く → 鉛筆アイコン（Edit）で中身を差し替え、または **Upload files** で同名アップロード
2. **Commit changes**
3. 数十秒でサイトに反映。**データはURLに紐づくのでそのまま残ります**

## バックアップについて
- 固定URLになってもバックアップ機能はそのまま使えます
- 大事なデータなので、ときどき「書き出し」でJSONの控えを取っておくと万全です
- 別端末へ移すときは、片方で「書き出し」→ もう片方で「読み込み」
